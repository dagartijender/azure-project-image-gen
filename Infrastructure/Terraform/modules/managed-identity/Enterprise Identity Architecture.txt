Production Best Practices
Use User Assigned Managed Identities (UAMI) instead of service principals wherever possible.
Enable AKS Workload Identity and federated identity credentials so Kubernetes service accounts can authenticate to Azure resources without secrets.
Apply least-privilege RBAC (for example, AcrPull, Storage Blob Data Contributor, Key Vault Secrets User, Azure Service Bus Data Sender/Receiver).
Create separate managed identities for distinct workloads (API, Worker, CI/CD, Application Gateway) to improve security isolation and simplify auditing.
Store no client secrets or connection strings in application configuration; use managed identities with Azure RBAC to access Azure services securely.
